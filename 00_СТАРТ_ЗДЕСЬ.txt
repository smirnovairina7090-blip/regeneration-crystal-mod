КРИСТАЛЛ РЕГЕНЕРАЦИИ - НАЧАТЬ ЗДЕСЬ
Minecraft 1.21.1 + NeoForge 21.1.x

САМОЕ ГЛАВНОЕ
Этот архив пока НЕ является готовым модом для TLauncher.
Внутри лежит проект с кодом, моделью и текстурой.
TLauncher принимает готовый файл с расширением .jar.

Путь такой:
1. Распаковать этот архив.
2. Загрузить содержимое проекта на GitHub.
3. GitHub Actions бесплатно соберёт файл .jar.
4. Скачать готовый .jar.
5. Установить его в сборку TLauncher с Minecraft 1.21.1 и NeoForge.

На компьютер не нужно устанавливать JDK 21, если сборка выполняется через GitHub Actions.

============================================================
ЭТАП 1. РАСПАКОВАТЬ АРХИВ
============================================================

1. Нажмите по ZIP правой кнопкой мыши.
2. Выберите «Извлечь всё».
3. Откройте распакованную папку.
4. В корне должны находиться, среди прочего:
   .github
   src
   blockbench_source
   build.gradle
   gradle.properties
   settings.gradle
   СТАРТ_ЗДЕСЬ.html

Не загружайте в GitHub сам ZIP одним файлом. Нужно загрузить содержимое распакованной папки.

============================================================
ЭТАП 2. СОЗДАТЬ РЕПОЗИТОРИЙ НА GITHUB
============================================================

1. Откройте github.com и войдите в аккаунт.
2. Нажмите кнопку New repository.
3. В поле Repository name напишите: regeneration-crystal-mod
4. Можно выбрать Public или Private.
5. Не добавляйте README, .gitignore и лицензию на этом экране.
6. Нажмите Create repository.

Что должно появиться:
Пустая страница нового репозитория с предложением загрузить файлы.

============================================================
ЭТАП 3. ЗАГРУЗИТЬ ПРОЕКТ
============================================================

1. На странице репозитория нажмите «uploading an existing file»
   или Add file -> Upload files.
2. Выделите ВСЁ содержимое распакованной папки и перетащите на страницу.
3. Дождитесь окончания загрузки.
4. Нажмите Commit changes.

ВАЖНАЯ ПРОВЕРКА
После загрузки в репозитории должны быть видны папки:
.github
src
blockbench_source

Откройте .github -> workflows.
Внутри должен быть файл build-mod.yml.
Без него облачная сборка не запустится.

============================================================
ЭТАП 4. СОБРАТЬ JAR НА GITHUB
============================================================

1. Откройте вкладку Actions в верхней части репозитория.
2. Слева выберите «Build Regeneration Crystal Mod».
3. Если сборка уже запустилась сама, просто дождитесь окончания.
4. Если она не запустилась, нажмите Run workflow, затем ещё раз Run workflow.
5. Откройте появившийся запуск.
6. Дождитесь зелёной галочки.

Что происходит в этот момент:
GitHub запускает Java 21 на своём сервере, скачивает NeoForge и Minecraft-зависимости,
проверяет код и собирает готовый мод.

Если появилась красная галочка:
1. Откройте неудачную сборку.
2. Откройте шаг Build mod.
3. Скопируйте текст ошибки или сделайте скриншот.
4. Не переходите в TLauncher, пока сборка не станет зелёной.

============================================================
ЭТАП 5. СКАЧАТЬ ГОТОВЫЙ JAR
============================================================

1. Откройте завершённую зелёную сборку.
2. Прокрутите страницу вниз до раздела Artifacts.
3. Нажмите regeneration-crystal-mod-jar.
4. GitHub скачает небольшой ZIP.
5. Распакуйте этот ZIP.
6. Внутри будет файл примерно такого вида:
   regeneration_crystal-1.0.0.jar

Именно этот .jar устанавливается в TLauncher.
Не устанавливайте в TLauncher:
- исходный ZIP проекта;
- папку src;
- файл .bbmodel;
- texture.png;
- build.gradle.

============================================================
ЭТАП 6. СОЗДАТЬ СБОРКУ В TLAUNCHER
============================================================

1. Запустите TLauncher.
2. Нажмите TL MODS в нижней правой части окна.
3. Нажмите Create в верхней части страницы.
4. Заполните поля:
   Название: Кристалл регенерации
   Загрузчик: NeoForge
   Версия Minecraft: 1.21.1
   Версия NeoForge: оставьте выбранную TLauncher версию ветки 21.1.x
5. Нажмите Create.
6. Убедитесь, что новая сборка выбрана.

Важно:
Нельзя выбирать Forge, Fabric или обычный Minecraft 1.21.1.
Для этого проекта нужен именно NeoForge и именно Minecraft 1.21.1.

============================================================
ЭТАП 7. ДОБАВИТЬ СВОЙ JAR В СБОРКУ TLAUNCHER
============================================================

СПОСОБ 1 - через ручную установку TL MODS
1. В TL MODS выберите сборку «Кристалл регенерации».
2. Нажмите кнопку с гаечным ключом и отвёрткой.
3. Выберите Manual installation / Ручная установка.
4. Нажмите кнопку выбора файла.
5. Выберите regeneration_crystal-1.0.0.jar.
6. Нажмите Install / Установить.

СПОСОБ 2 - через папку mods
Используйте его, если пункта ручной установки нет.
1. Выберите созданную сборку в TLauncher.
2. Нажмите значок папки или «Открыть папку».
3. Найдите папку mods.
4. Если её нет, создайте папку с названием mods.
5. Скопируйте туда regeneration_crystal-1.0.0.jar.

В папке mods должен лежать именно файл .jar.

============================================================
ЭТАП 8. ЗАПУСТИТЬ И ПРОВЕРИТЬ
============================================================

1. В TLauncher выберите сборку «Кристалл регенерации».
2. Проверьте, что возле неё указаны Minecraft 1.21.1 и NeoForge.
3. Нажмите «Войти в игру».
4. Дождитесь главного меню Minecraft.
5. Откройте список модов и найдите Regeneration Crystal.
6. Создайте новый мир в творческом режиме.
7. Откройте инвентарь и введите в поиске:
   Кристалл регенерации
8. Поставьте блок на землю.
9. Подойдите на расстояние до 5 блоков.
10. Нажмите по кристаллу правой кнопкой мыши.

Ожидаемый результат:
- кристалл отображается с авторской моделью и текстурой;
- блок светится;
- игроки в радиусе 5 блоков получают регенерацию на 10 секунд;
- появляется сообщение об активации.

============================================================
ЧТО ЛЕЖИТ В ПРОЕКТЕ И ЧТО ПОКАЗЫВАТЬ НА УРОКЕ
============================================================

1. blockbench_source/regeneration_crystal.bbmodel
   Исходная модель, созданная в Blockbench.
   Показываем, как визуальный объект хранится отдельно от Java-кода.

2. blockbench_source/regeneration_crystal_export.json
   Экспорт геометрии модели в формате, который понимают ресурсы Minecraft.

3. src/main/resources/assets/regeneration_crystal/textures/block/regeneration_crystal.png
   Текстура кристалла.
   Показываем, что PNG отвечает за внешний вид граней модели.

4. src/main/resources/assets/regeneration_crystal/models/block/regeneration_crystal.json
   Модель блока для Minecraft.
   В этом JSON описана геометрия и указана текстура.

5. src/main/resources/assets/regeneration_crystal/blockstates/regeneration_crystal.json
   Связывает состояние блока с его моделью.

6. src/main/resources/assets/regeneration_crystal/lang/ru_ru.json
   Русское название блока в игре.

7. src/main/resources/data/regeneration_crystal/recipe/regeneration_crystal.json
   Рецепт создания кристалла.

8. src/main/resources/data/regeneration_crystal/loot_table/blocks/regeneration_crystal.json
   Определяет, что выпадет при разрушении блока.

9. src/main/java/ru/creativika/regenerationcrystal/RegenerationCrystalMod.java
   Главный Java-класс мода.
   Показываем MOD_ID, импорты и подключение регистраций.

10. src/main/java/ru/creativika/regenerationcrystal/registry/ModBlocks.java
    Регистрация блока.
    Здесь Minecraft узнаёт, что новый блок существует.

11. src/main/java/ru/creativika/regenerationcrystal/registry/ModItems.java
    Регистрация предмета блока, который можно держать в инвентаре.

12. src/main/java/ru/creativika/regenerationcrystal/event/CrystalEvents.java
    Логика действия кристалла.
    Здесь проверяются нажатие, расстояние и выдача эффекта регенерации.

13. src/main/resources/META-INF/neoforge.mods.toml
    Паспорт мода: название, версия, загрузчик и совместимость.

14. build.gradle, settings.gradle, gradle.properties
    Настройки сборки проекта.
    Их не нужно переносить в TLauncher. Они нужны GitHub, чтобы получить JAR.

15. .github/workflows/build-mod.yml
    Инструкция для GitHub Actions.
    Она запускает Java 21, Gradle и создаёт артефакт с готовым JAR.

============================================================
ЧАСТЫЕ ОШИБКИ
============================================================

1. В TLauncher загрузили исходный ZIP.
   Решение: нужен JAR из раздела GitHub Actions -> Artifacts.

2. Выбрана неправильная версия Minecraft.
   Решение: использовать 1.21.1.

3. Выбран Forge вместо NeoForge.
   Решение: создать сборку с загрузчиком NeoForge.

4. JAR лежит не в той сборке.
   Решение: выбрать нужный модпак перед ручной установкой или открыть его папку.

5. GitHub Actions не появился.
   Решение: проверить наличие .github/workflows/build-mod.yml в репозитории.

6. Сборка GitHub красная.
   Решение: открыть Build mod и прислать текст ошибки.

7. Игра запускается, но блока нет.
   Решение: проверить раздел Mods, версию загрузчика и наличие JAR в папке mods.

============================================================
